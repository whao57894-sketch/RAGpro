# 技术选型说明

## 后端框架：FastAPI

选择 FastAPI 作为 API 服务框架。它支持异步接口、自动生成 OpenAPI 文档，适合实现文档上传、问答请求、知识库管理等接口。

## 大模型：智谱 GLM

选择智谱 API 作为第一版大模型能力。原因是国内访问稳定，SDK 使用成本低，适合企业文档问答场景的快速验证。

## RAG 编排：LangChain

选择 LangChain 负责文档加载、文本切分、检索链路和模型调用编排。第一版可以快速搭建 RAG 流程，后续如果性能或可控性不足，可逐步替换为自定义链路。

## PDF 解析：pypdf

选择 `pypdf` 处理文本型 PDF。它轻量、依赖少，适合规章制度、产品手册等常见文本 PDF。对于扫描件、图片型 PDF，后续需要引入 OCR。

## 向量数据库：ChromaDB

选择 ChromaDB 作为本地向量库候选。它支持持久化、写入和相似度检索，适合单机原型和面试项目展示。

当前验证结论：ChromaDB 本身适合项目需求。Python 3.13 环境安装 `chromadb==0.6.3` 会遇到底层编译依赖问题；切换到 Python 3.11.9 后已成功安装，并完成向量写入和检索验证。

## 环境建议

- Python：使用 3.11.9。
- 依赖安装位置：项目本地 `.venv`。
- 配置管理：使用 `.env` 保存 API Key、模型名、向量库路径等配置。
- 数据目录：上传文件和向量库统一放在 `data/` 下，便于本地调试和清理。
